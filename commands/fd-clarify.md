# /fd-clarify

Trigger the **frontend-design** skill in **clarify mode**.

Use this command when you want to:
- Improve vague or generic labels and button text
- Rewrite unhelpful error messages into actionable guidance
- Add meaningful empty states and placeholder text
- Make microcopy specific, helpful, and human

## How to Use

```
/fd-clarify
```

Examples:
- `/fd-clarify` on a form with generic "Submit" buttons and "Error occurred" messages
- `/fd-clarify` on empty states that show only blank space

## What Happens

1. The frontend-design skill activates in clarify mode
2. The ux-writing reference is loaded for microcopy best practices
3. All labels, buttons, error messages, and empty states are reviewed
4. Error messages are rewritten following the 3-part pattern: what happened, why, what to do
5. Button labels are made specific to their action
6. Changes are applied and verified

## Output

Rewritten microcopy throughout the UI — specific button labels, actionable error messages, and meaningful empty states.
