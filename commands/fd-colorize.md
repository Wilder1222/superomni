# /fd-colorize

Trigger the **frontend-design** skill in **colorize mode**.

Use this command when you want to:
- Replace a generic or monochrome palette with strategic color
- Apply an OKLCH-based color system with tinted neutrals
- Establish proper color ratios and dark mode support
- Create visual distinction through intentional color use

## How to Use

```
/fd-colorize
```

Examples:
- `/fd-colorize` on a grayscale dashboard that needs visual energy
- `/fd-colorize` on a landing page with default blue links and no color identity

## What Happens

1. The frontend-design skill activates in colorize mode
2. The color-and-contrast reference is loaded for palette best practices
3. An OKLCH-based palette is generated with tinted neutrals
4. The 60-30-10 color ratio is applied across the interface
5. Dark mode support is added with proper color mapping
6. Contrast ratios are verified against WCAG standards
7. Changes are applied and verified

## Output

A cohesive, OKLCH-based color system applied across the UI with tinted neutrals, proper ratios, and dark mode support.
