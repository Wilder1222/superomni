# /fd-critique

Trigger the **frontend-design** skill in **critique mode**.

Use this command when you want to:
- Evaluate information architecture and content structure
- Assess visual hierarchy and element prioritization
- Review user flow clarity and navigation paths
- Identify confusion points in the interface

## How to Use

```
/fd-critique
```

Examples:
- `/fd-critique` on a dashboard to assess hierarchy and scanability
- `/fd-critique` on a multi-step form to evaluate user flow clarity

## What Happens

1. The frontend-design skill activates in critique mode
2. Layout hierarchy and visual weight distribution are analyzed
3. Confusion points and ambiguous UI patterns are identified
4. Information architecture is evaluated against user goals
5. Specific improvements are suggested with rationale
6. Changes are applied and verified

## Output

A UX critique with identified issues and applied improvements to hierarchy, flow, and clarity.
